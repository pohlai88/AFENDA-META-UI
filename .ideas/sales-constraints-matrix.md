# Sales Constraints Matrix

Date: March 27, 2026
Scope: High-value sales tables with business-critical DB enforcement

---

## Purpose

This matrix centralizes how business rules are enforced at the database level for the sales domain.
It maps each critical invariant to concrete enforcement primitives: CHECK constraints, UNIQUE indexes, foreign keys, and tenant isolation policies.

Primary schema source: `packages/db/src/schema-domain/sales/tables.ts`

---

## Notes

1. This matrix focuses on structural and integrity enforcement at the DB layer.
2. Application/service invariants should complement, not replace, these constraints.
3. New high-risk tables should add an entry here in the same change set as schema updates.

## 2. How does V8 coverage compare to METRICS.md?

**Two different measurement types:**

### V8 Coverage (Vitest Provider)

**What it measures:** Code execution during tests

| Metric         | Current | Target | What It Means            |
| -------------- | ------- | ------ | ------------------------ |
| **Lines**      | 43.65%  | 80%    | % of code lines executed |
| **Branches**   | 36.92%  | 75%    | % of if/else paths taken |
| **Functions**  | 51.56%  | 85%    | % of functions called    |
| **Statements** | 43.64%  | 80%    | % of statements executed |

**Generated by:** `vitest --coverage` (uses V8 JavaScript engine)
**Reports:** `coverage/coverage-summary.json`, `coverage/index.html`
**Config:** `vitest.config.ts` → `coverage.thresholds`

### Truth-Grade Metrics (METRICS.md)

**What it measures:** Business logic validation

| Metric                  | Current | Target | What It Means                                |
| ----------------------- | ------- | ------ | -------------------------------------------- |
| **Invariant Coverage**  | TBD     | 100%   | % of invariants tested (positive + negative) |
| **Transition Coverage** | TBD     | 100%   | % of state machine transitions tested        |
| **Mutation Coverage**   | TBD     | 100%   | % of CRUD operations tested per entity       |
| **Event Coverage**      | TBD     | 100%   | % of domain events verified                  |

**Measured by:** Manual analysis + custom tooling (planned)
**Reports:** METRICS.md (manual updates)
**Config:** METRICS.md defines what to count

### Key Differences

```
V8 Coverage: "Did this line of code run during tests?"
Truth-Grade: "Did we test this business rule?"
```

**Example:**

```typescript
// This function has 95% V8 line coverage
function evaluateCondition(expr, record) {
  if (expr.logic === "and") {
    return expr.conditions.every((c) => check(c, record));
  }
  return expr.conditions.some((c) => check(c, record));
}

// But only 2/15 invariants are tested
// → V8 Coverage: 95%
// → Truth-Grade Invariant Coverage: 13%
```

**Both are needed:**

1. **V8 Coverage** ensures test infrastructure code works
2. **Truth-Grade** ensures business rules are validated
